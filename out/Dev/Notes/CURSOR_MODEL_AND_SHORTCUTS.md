# Cursor 2.0 — 모델 · 단축키 참고 (2026)

## 1. 모델 설정

- **UI**: Composer/Chat → 오른쪽 모델 드롭다운 → 선택
- **기본 고정**: Settings → Models → 기본 모델 Composer 1.5
- **고급**: `.cursor/settings.json`에 아래 적용됨
  ```json
  {
    "models": {
      "default": "composer-1.5",
      "codebase_indexing": "gemini-3-pro"
    }
  }
  ```

**실전 팁**

- 계획 단계 → Claude Opus
- 실행 단계 → Composer 1.5
- 작업 중간에 모델 바꾸지 말 것 (컨텍스트 깨짐)

---

## 2. 핵심 단축키 (Windows)

| 기능                         | 단축키               | 용도 예시                                 |
| ---------------------------- | -------------------- | ----------------------------------------- |
| Inline Edit (한 파일 수정)   | `Ctrl + K`         | "이 함수를 async로 바꿔줘"                |
| Chat (질문/설명)             | `Ctrl + L`         | "이 코드 왜 에러나?"                      |
| Composer (멀티파일 에이전트) | `Ctrl + I`         | "크롤링 → Notion 자동화 스크립트 만들어" |
| Composer Full                | `Ctrl + Shift + I` | 전체 프로젝트 리팩토링                    |
| Add to Context               | `Ctrl + Shift + L` | 코드 선택 후 "이 파일도 고려해서"         |
| 모델 순환 전환               | `Ctrl + Shift + J` | 빠르게 모델 변경                          |
| 설정 열기                    | `Ctrl + ,`         | 모델/단축키 커스텀                        |

**프로 팁**

- `Ctrl + K` = 한 파일, `Ctrl + I` = 프로젝트 전체
- 단축키 변경: `Ctrl + Shift + P` → "Keyboard Shortcuts" → 원하는 명령 remap

## 3. 실제 사용 가능 여부

| 항목                                             | 사용 가능         | 비고                                                                                                                              |
| ------------------------------------------------ | ----------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| **단축키** (Ctrl+K, L, I 등)               | ✅ 바로 사용      | Cursor 기본 키 바인딩. 변경 안 했다면 그대로 동작                                                                                 |
| **모델 드롭다운 / Settings → Models**     | ✅ 사용           | 앱 UI에서 제공. 표시되는 모델 이름은 구독·버전에 따라 다를 수 있음                                                               |
| **`.cursor/settings.json`**              | ⚠️ 앱 따라 다름 | Cursor가 이 경로의 `models`를 읽는지는 버전별로 다를 수 있음. 적용 안 되면 **Settings → Models**에서 직접 기본 모델 선택 |
| **실전 팁** (계획→Claude, 실행→Composer) | ✅ 적용 가능      | 작업 흐름 권장사항                                                                                                                |
